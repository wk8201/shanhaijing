# 部署验证与线上访问测试指南

## 验证 GitHub Pages 部署

一旦 GitHub Pages 部署完成，您可以按照以下步骤验证部署并测试线上访问：

### 1. 检查部署状态

1. 登录 GitHub 并进入您的"sh-ancient"仓库
2. 点击顶部导航栏中的"Settings"（设置）选项卡
3. 在左侧菜单中找到并点击"Pages"选项
4. 查看页面顶部是否显示绿色的成功消息，表明部署成功
5. 复制 GitHub 提供的网站 URL，格式通常为：`https://[您的GitHub用户名].github.io/sh-ancient/`

### 2. 测试网站访问

1. 打开浏览器，粘贴复制的 URL
2. 验证网站是否正常加载
3. 测试网站的基本功能，包括：
   - 导航菜单是否正常工作
   - 页面内容是否完整显示
   - 图片、样式和脚本是否正确加载
   - 响应式设计在不同设备尺寸下是否正常

### 3. 检查路由功能（重要）

由于我们使用了 Vue Router，如果您的网站有多个路由页面，请确保：

1. 直接访问特定路径（如 `/about`）是否正常工作
2. 刷新页面后，网站是否仍然能够正确加载当前路由
3. 如果遇到路由问题，可能需要检查：
   - GitHub Pages 设置是否正确
   - Vue Router 的 base 路径配置
   - 可能需要添加 404.html 文件以支持 HTML5 History 模式

## 常见问题排查

### 1. 网站无法访问

- 确认部署是否完成（可能需要几分钟时间）
- 检查 URL 是否正确
- 验证 GitHub Pages 设置是否正确配置

### 2. 样式或脚本不加载

- 检查浏览器控制台是否有错误信息
- 确认资源路径是否正确（可能与 base 路径配置有关）
- 验证 vite.config.ts 中的 base 配置是否与仓库名称匹配

### 3. 路由问题（刷新页面 404）

- 对于使用 HTML5 History 模式的 Vue 应用，您可能需要创建一个自定义的 404.html 文件，将所有请求重定向回 index.html
- 或者考虑使用 Hash 模式路由：在 router 配置中使用 createWebHashHistory()替代 createWebHistory()
