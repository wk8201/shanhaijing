# 后续部署操作指南

当您需要更新并重新部署项目时，可以按照以下步骤操作：

## 1. 本地开发与更新

1. **在本地进行代码修改**

   - 根据需求修改源代码
   - 添加新功能或修复 bug

2. **测试修改**
   - 运行开发服务器：`npm run dev`
   - 确保所有功能正常工作
   - 测试不同浏览器和设备的兼容性

## 2. 构建与验证

1. **构建项目**

   ```bash
   npm run build
   ```

   - 这将生成更新后的静态文件到`dist`目录

2. **验证构建结果**（可选但推荐）
   - 您可以使用简单的静态文件服务器测试构建结果
   - 例如使用`http-server`：
     ```bash
     npm install -g http-server
     cd dist
     http-server
     ```
   - 访问本地服务器地址，确保网站正常运行

## 3. 代码提交与推送

1. **提交更改到 Git 仓库**

   ```bash
   git add .
   git commit -m "描述您的更新内容"
   git push origin master  # 或main分支
   ```

2. **自动部署触发**
   - 如果您已经配置了 GitHub Actions（.github/workflows/deploy.yml），推送代码后将自动触发部署
   - GitHub Actions 会自动构建项目并部署到 GitHub Pages

## 4. 监控部署进度

1. **查看部署状态**

   - 登录 GitHub 并进入您的仓库
   - 点击顶部的"Actions"选项卡
   - 查看最新的工作流运行状态
   - 点击具体运行可查看详细日志

2. **确认部署完成**
   - 部署成功后，工作流状态将显示为绿色的"Success"
   - 您可以通过 GitHub Pages 设置页面查看网站 URL

## 5. 验证线上更新

1. **访问线上网站**

   - 打开浏览器，访问您的 GitHub Pages 网站 URL
   - 可能需要清除浏览器缓存以查看最新更改

2. **功能测试**
   - 验证所有更新是否正确显示
   - 测试新增功能是否正常工作

## 常见问题处理

如果在更新部署过程中遇到问题：

1. **构建失败**

   - 检查构建错误信息
   - 确保所有依赖已正确安装
   - 解决代码中的错误

2. **部署失败**

   - 查看 GitHub Actions 日志找出具体错误
   - 常见问题包括 Node.js 版本不兼容、依赖安装失败等

3. **网站不显示更新**
   - 清除浏览器缓存
   - 确认 GitHub Pages 已完成部署
   - 检查是否推送到了正确的分支

## 注意事项

1. **备份重要数据**：在进行重大更新前，建议备份当前代码
2. **版本控制**：使用有意义的提交信息，便于跟踪更改
3. **定期更新依赖**：定期运行`npm update`确保依赖包保持最新
4. **监控性能**：关注网站性能变化，及时进行优化

通过以上步骤，您可以轻松地更新并重新部署您的 Vue 项目到 GitHub Pages。
